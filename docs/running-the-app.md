# Building and running the mobile app

The app is an Expo SDK 56 / React Native 0.85 project living entirely in `frontend/`. It uses native modules (MapLibre Native, Skia, MMKV), so it **cannot run in Expo Go** — you build a dev client with `expo run:ios` / `expo run:android` (wrapped by the `bun run` scripts below).

## Prerequisites

| tool | version | notes |
|---|---|---|
| [Bun](https://bun.sh) | 1.1+ | replaces npm/yarn — all scripts assume `bun` |
| JDK | 17 | Gradle requirement (Android) |
| Android Studio | with API 35 SDK + emulator image | Android builds |
| Xcode | 26+ | iOS — deployment target is 26.0, macOS only |
| CocoaPods | 1.15+ | `gem install cocoapods` (iOS) |
| Watchman | latest | Metro file watching on macOS |

You only need the toolchain for the platform you're targeting. Linux hosts can build/run Android only.

## Run

All commands from `frontend/` — not the repo root:

```bash
cd frontend
bun install
bun start            # Metro bundler only (attach to an already-installed dev build)
bun run android      # build + install + launch on Android emulator/device
bun run ios          # build + install + launch on iOS simulator (macOS only)
```

First native build takes a while (Gradle / CocoaPods); subsequent runs are incremental.

## Android gotchas

- Use an **API 35** emulator image.
- The emulator cannot see `localhost` on your machine — reach a backend on the host via **`http://10.0.2.2:8080`** (the emulator's loopback alias).
- Predictive back gesture is deliberately disabled (it fights the bottom sheet). Don't re-enable it in `app.json` without retesting the sheet.

## iOS gotchas

- Boot a simulator first: `xcrun simctl list devices booted`
- Grant location: `xcrun simctl privacy booted grant location com.vanillax.radar-ng`
- **Never hand-edit `frontend/ios/`** — it's generated by `expo prebuild` and gitignored. Native customization goes through config plugins in `frontend/plugins/` and target sources in `frontend/targets/`.
- The CarPlay config plugin (`frontend/plugins/withCarPlayScene.js`) declares both the phone and CarPlay scenes — keep both.

## Connecting to a backend

In the app: **Settings → Data Source**.

- **Free tier** (default, zero setup): IEM NEXRAD radar tiles + public Open-Meteo + NWS alerts. Works out of the box, fewer layers, 5-min radar.
- **Self-Hosted**: point at your own stack ([self-hosting.md](self-hosting.md)) and enter the URL:

| where the app runs | server URL |
|---|---|
| iOS simulator (backend on same Mac) | `http://localhost:8080` |
| Android emulator | `http://10.0.2.2:8080` |
| Physical device on LAN | `http://<your-LAN-IP>:8080` |
| Public exposure | `https://radar.your-domain.example` |

The manifest fetch (`/api/manifest.json`) determines which layers the app offers, so a partially warmed-up backend just shows fewer layers until ingest catches up.

## Where preferences live

User prefs (server URL, data source, palette, opacity, active layer, theme, extras visibility) persist in an **MMKV** store with id `radar-ng` (`frontend/src/lib/storage.ts`), read/written by `frontend/src/stores/useWeatherStore.ts`. They survive app restarts but not an uninstall. To reset to defaults, uninstall/reinstall or clear the app's storage.

## iOS extras: CarPlay + Apple Watch

The repo ships a CarPlay map scene and a watchOS forecast app as extra Apple targets (`frontend/targets/{carplay,watch}/`, copied into `ios/` on prebuild). These are **personal-sideload only** — CarPlay in particular needs an entitlement Apple won't grant for non-navigation apps, so a re-signing workaround is required for real-car use (the CarPlay *Simulator* needs no hacks).

Full walkthrough, signing hack included: [carplay-watch-setup.md](carplay-watch-setup.md).
